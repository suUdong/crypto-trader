# CI Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** crypto-trader에 GitHub Actions 기반 최소 CI 파이프라인을 도입해 매매 코어 회귀를 PR/master push 시점에 게이트한다.

**Architecture:** 단일 워크플로우 파일 (`.github/workflows/ci.yml`), 단일 Job, 단일 Python 3.12. ruff (필수) → mypy (continue-on-error) → pytest (화이트리스트) 순차 실행. 코드 변경 최소화 — 테스트 마커/디렉토리 이동 없이 워크플로우에 화이트리스트 직접 나열. duckdb 한 줄만 메인 deps에 추가.

**Tech Stack:** GitHub Actions, Python 3.12, pip, ruff, mypy, pytest, pyproject.toml.

**Spec:** `docs/superpowers/specs/2026-04-07-ci-setup-design.md`

**Branch:** `feature/ci-setup` (이미 생성됨, 디자인 스펙 + .gitignore 핫픽스 커밋됨)

- [ ] **Step 2: Write `.github/workflows/ci.yml`**

아래 내용을 그대로 작성. 단, `pytest` 라인의 화이트리스트는 Task 2 Step 6에서 기록한 **최종 화이트리스트**로 교체 (실패 제외 후 버전).

```yaml
name: ci

on:
  pull_request:
  push:
    branches: [master]

permissions:
  contents: read

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  test:
    name: lint + type + test
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -e ".[dev]"

      - name: Lint (ruff)
        run: ruff check src/ tests/ scripts/

      - name: Type check (mypy, warnings only)
        run: mypy src/
        continue-on-error: true

      - name: Test (whitelist)
        run: |
          pytest \
```
